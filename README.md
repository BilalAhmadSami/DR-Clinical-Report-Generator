# Diabetic Retinopathy Clinical Report Generator

A multimodal medical-AI pipeline: a trained computer-vision model grades
diabetic-retinopathy severity from a retinal fundus image, and a vision-language
LLM (Gemini) drafts a structured clinical report around that grade.

> WARNING: Research and education demo only. NOT for clinical use. The grade is
> produced by an automated model and the report is AI-generated; both must be
> reviewed by a qualified ophthalmologist. This is not a medical device.

## What it does

1. You upload a retinal fundus image.
2. A trained EfficientNet-B4 + Swin Transformer model (CORN ordinal head) grades
   it 0 to 4 (No DR, Mild, Moderate, Severe, Proliferative).
3. The grade plus the image are sent to Gemini, which writes a structured report:
   clinical interpretation, observable features, recommended follow-up, and
   caveats about automated grading.
4. The app shows the grade, a per-class severity bar, the preprocessed image, and
   the report.

## Why this design

A general vision-language model is not validated to diagnose retinal images, so
the **diagnosis stays with the specialist model** (which scored Test QWK ~0.81 on
held-out data) and the **LLM only writes the report** around that fixed grade.
This separation of "seeing/grading" from "writing" is what keeps the pipeline
defensible. The grading model is reused from the
[Diabetic Retinopathy Grading](https://github.com/BilalAhmadSami/Diabetic-Retinopathy-Grading)
project.

## Architecture

```
fundus image
   |
   v
preprocessing (auto fundus crop -> pad -> resize 384 -> CLAHE)   [matches training]
   |
   v
EfficientNet-B4 + Swin Transformer (parallel) -> CORN ordinal head -> grade 0-4
   |
   v
grade + image -> Gemini (vision-language LLM) -> structured clinical report
```

## Stack

| Component | Choice |
|-----------|--------|
| Grading model | EfficientNet-B4 + Swin-B384, CORN ordinal loss (PyTorch, timm) |
| Preprocessing | OpenCV (fundus crop + CLAHE, matched to training) |
| Report LLM | Google Gemini (Flash) via the free AI Studio API |
| UI | Gradio |
| Weights | downloaded from the public HuggingFace Space at first run |

## Setup and run

```bash
# 1. Install dependencies (CPU is fine; this runs single-image inference)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cpu
pip install -r requirements.txt

# 2. Add your free Gemini key
cp .env.example .env        # then edit .env and paste your key from https://aistudio.google.com

# 3. Run the app
python app.py               # opens a local Gradio URL
```

The model weights (about 430 MB) download automatically on first run and are
cached afterwards.

## Limitations

- The grading model reflects its training data and is not a certified diagnostic
  tool; performance on out-of-distribution images will be lower.
- The report is generated by a general LLM constrained to the model's grade; it
  is illustrative, not a real clinical document, and can phrase findings
  generically.
- For research and education only.
